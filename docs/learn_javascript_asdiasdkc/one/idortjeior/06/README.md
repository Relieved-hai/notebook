# Intl：JavaScript中的国际化

JavaScript中的字符串，日期和数字的一个常见问题是，它们"不知道"访问者所在国家/地区的语言和特殊性。

特别是：

<br/>

**字符串**

比较时会比较字符代码，这是错误的，例如，在俄语中，结果是 `"ё" > "я"` 和 `"а" > "Я"`，，尽管每个人都知道这 `я` 是字母表的最后一个字母，并且必须大于其他任何一个。

**日期**

不同的国家/地区对日期使用不同的表示法。他们在2014年12月31日（俄罗斯）写的某个地方，在2014年12月31日（美国）写的某个地方的某个地方。

**号码**

在某些国家/地区，它们以数字显示，而在其他国家/地区，以象形文字显示，长数字在某个地方用空格隔开，在某些地方用逗号隔开。

<br/>

除IE10（但有库）以外的所有现代浏览器都支持 [ECMA 402](http://www.ecma-international.org/ecma-402/1.0/ECMA-402.pdf) 标准，该标准旨在永久解决这些问题。
