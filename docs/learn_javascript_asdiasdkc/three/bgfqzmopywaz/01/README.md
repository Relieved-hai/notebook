# 关于

我们现在真的要学习 CSS 吗？没什么假定您已经了解 CSS，或者至少在允许您创建Web页面的水平上了解 CSS。

JavaScript 开发人员资格的特殊性在于，他没有义务选择颜色，绘制图标，“做得漂亮”。除非他是兼职版图专家，否则他也没有义务用 HTML 排版版式。

**那是他绝对必须要做的-为不中断的控件开发这样的 HTML/CSS 结构，并便于以后与他进行交互。**

这需要在定位元素领域具有丰富的CSS知识，包括复杂的 `display`, `margin`, `border`, `outline`, `position`, `float`, `border-box` 和其他属性，以及构建组件结构的方法（CSS布局） ）。

JavaScript 做很多事情。通常，在不了解CSS的情况下，你会这样做，但是我们不会深入其中。

```
如果可以通过CSS完成某些操作，则最好通过CSS进行。
```

原因很简单-通常，即使 CSS 看起来更复杂-与 JS 相比，它更易于维护和开发。 因此，值得一试。

此外，还有另一项观察。

```
JavaScript的知识不能替代CSS的知识。
```

如果您对CSS和JavaScript都有一定的了解，会变得更加轻松愉快。

## 清单

以下是“清单”。 如果至少有一个不熟悉的属性，这是进一步阅读本节的信号灯。

- 块模型包括：
  - `margin`, `padding`, `border`, `overflow`
  - 以及 `height/width` 和 `min-height/min-width`。

- 文本：

  - `font`
  - `line-height`.

- 各种游标 `cursor`。

- 定位：

  - `position`, `float`, `clear`, `display`, `visibility`
  - 以 CSS 为中心
  - 重叠的 `z-index` 和 `opacity`

- 选择器：

  - 选择器优先级
  - 选择器 `＃id`，`.class`，`a > b`

- 重置浏览器样式 [reset.css](http://meyerweb.com/eric/tools/css/reset/)
